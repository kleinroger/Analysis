# 项目结构

```
├─app/
├─migrations/
├─tests/
├─static/
├─templates/
├─docs/
├─requirements/
├─tools/
├─.env/
└─README.md
```

## 目录用途
- `app/`：应用源码与模块
- `migrations/`：数据库迁移脚本
- `tests/`：测试用例与基准
- `static/`：静态资源（CSS/JS/图片等）
- `templates/`：页面模板或服务端渲染模板
- `docs/`：项目文档与设计说明
- `requirements/`：依赖管理（如 `requirements.txt` 等）
- `tools/`：开发/运维工具脚本
- `.env/`：环境变量相关文件（本地私有配置）

## 使用提示
- 每个目录已创建占位文件以便版本控制跟踪；随后可按需替换为真实内容。
